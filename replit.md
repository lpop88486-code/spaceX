# SpaceXStarlink

A production-ready satellite internet service website inspired by Starlink, with full Stripe payment integration and an admin dashboard.

## Run & Operate

- `pnpm --filter @workspace/api-server run dev` — run the API server (port 8080)
- `pnpm --filter @workspace/spacex-starlink run dev` — run the frontend (port 20040)
- `pnpm run typecheck` — full typecheck across all packages
- `pnpm run build` — typecheck + build all packages
- `pnpm --filter @workspace/api-spec run codegen` — regenerate API hooks and Zod schemas from the OpenAPI spec
- `pnpm --filter @workspace/db run push` — push DB schema changes (dev only)
- Required env: `DATABASE_URL` — Postgres connection string
- Optional env: `STRIPE_SECRET_KEY` — Stripe secret key (app works in mock mode without it)
- Optional env: `STRIPE_WEBHOOK_SECRET` — Stripe webhook secret for payment confirmations
- Optional env: `ADMIN_PASSWORD` — Admin dashboard password (default: admin123)
- Optional env: `SESSION_SECRET` — JWT signing secret

## Stack

- pnpm workspaces, Node.js 24, TypeScript 5.9
- Frontend: React + Vite, TailwindCSS, shadcn/ui, framer-motion, Recharts
- API: Express 5
- DB: PostgreSQL + Drizzle ORM
- Payments: Stripe (checkout sessions + webhooks)
- Auth: JWT-based admin auth
- Validation: Zod (`zod/v4`), `drizzle-zod`
- API codegen: Orval (from OpenAPI spec)
- Build: esbuild (CJS bundle)

## Where things live

- `lib/api-spec/openapi.yaml` — OpenAPI source of truth
- `lib/db/src/schema/` — DB schema (plans.ts, subscriptions.ts)
- `artifacts/api-server/src/routes/` — Express route handlers
- `artifacts/spacex-starlink/src/` — React frontend
- `lib/api-client-react/src/generated/` — Generated React Query hooks
- `lib/api-zod/src/generated/` — Generated Zod schemas

## Architecture decisions

- Stripe is optional — the app runs in "mock mode" if `STRIPE_SECRET_KEY` is not set, creating subscriptions directly as "active" without redirecting to Stripe
- Admin auth uses simple JWT tokens stored in localStorage; admin password is set via `ADMIN_PASSWORD` env var
- The `api-zod` index only re-exports from `./generated/api` (not `./generated/types`) to avoid naming conflicts from Orval's split mode
- Stripe webhook secret is optional — webhooks still work in dev without signature verification

## Product

- Public landing page with cinematic starfield animation
- Service plans page: Residential, Roam, and Business tiers
- Stripe checkout flow for subscribing to plans
- Admin dashboard (login at /admin) with stats, subscription management, and plan CRUD

## User preferences

- Admin dashboard at /admin/dashboard
- Default admin password: admin123 (change with ADMIN_PASSWORD env var)

## Gotchas

- Always run `pnpm --filter @workspace/db run push` after changing the DB schema
- After changing `lib/api-spec/openapi.yaml`, run codegen before touching backend types
- The `lib/api-zod/src/index.ts` file is regenerated by codegen — keep it as `export * from "./generated/api"` only
- Stripe: use test mode keys for development. Set `STRIPE_SECRET_KEY` to enable real payments.
